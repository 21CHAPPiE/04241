## 总体说明

当前系统中的大模型不直接承担水文数值计算，而是承担“任务理解与工具组织者”的角色。完整执行逻辑为：

- 接收自然语言请求
- 识别任务类型与关键参数
- 调用 FastMCP server 中注册的 tools
- 显式执行数据加载接口
- 显式执行模型资产加载接口
- 依次执行预测、集成、修正等工具链
- 汇总中间结果和最终结果
- 输出结构化场景报告

本轮实际参与执行的 FastMCP tools 包括：

- `dataset_profile_from_paths`
- `model_asset_profile`
- `train_model_bundle_from_paths`
- `forecast_from_paths`
- `data_analysis_from_paths`
- `ensemble_from_paths`
- `correction_from_paths`

相较于旧版本，当前架构已经完成两项关键调整：

- 所有工具注册都通过独立 `fastmcp` 库实现
- benchmark 场景通过 FastMCP server 实例调用工具，而不是直接调用 Python 函数

该场景说明，大模型已经能够通过 FastMCP 工具链识别“更新任务”，并在重新加载更新数据窗口后重跑预测主线、输出新旧结果对比。

## 当前边界

当前结果也反映出几个明确边界：

- 复杂洪水样本目前仍使用仓库内 `2016-09-28` 高流量窗口做代理演示，不是计划中的 `2024061623` 实际过程。
- 模型筛选信息已经可输出，但当前没有额外设置阈值，因此 `screening` 结果仍为全模型通过。
- 滚动更新当前属于“扩展观测窗后的重跑与比较”，还不是严格的增量式状态续算引擎。
