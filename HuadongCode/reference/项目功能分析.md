# 第一阶段项目评估稿

## 一、项目问题定型

### 1. 问题定义

建议将该项目定义为“预测驱动的滚动水库调度与水资源配置决策支持问题”，即在既定来水预报预见期、工程规则与边界约束下，形成可回放评估、可比选、可解释的预报结果、风险评估与可执行的出库及配水方案，并能触发干旱与洪水预警。

- 这样定义的原因：用户需求同时包含预测、预警与调度算法，但真正业务闭环通常以“调度决策可落地”为终点；预测与风险分析属于上游驱动能力，误差解释与异常识别属于质量与可解释性保障能力。

备选定义：建议按“两条主线并行”的工程问题处理，即“滚动水文预报服务”与“规则约束下的滚动水库调度服务”，在第一阶段通过统一数据口径与接口契约耦合，而非一开始做全量一体化最优联合求解。

### 2. 问题分类

- 主类别：调度问题  
- 次类别：预测问题、监测预警问题、仿真评估问题、优化问题、规则整编问题  
- 分类依据：需求同时出现“多尺度径流预测与集合预测”“干旱/洪水预警触发”“水资源配置与水量/水库调度算法”“风险概率统计与误差识别解释”，符合“预测驱动+规则约束+滚动出库计划”的典型组合；但第一阶段不宜默认上多库联合全局优化或复杂黑盒智能算法，应先白盒化规程规则、跑通滚动仿真与服务调用。

### 3. 问题拆解

- 业务目标：在防洪与兴利目标下，形成可执行调度与配水方案；对预测结果提供质量评估、风险刻画与预警触发；支持业务复盘与会商比选。
- 决策对象：单库或多库水库群；关联供水对象与下游控制断面；相关站点与流域的来水预报对象。
- 状态量：库水位、库容、入库流量、出库流量、闸门机组状态、下游控制断面水位流量、供需缺口指标、旱涝指标、模型误差状态。
- 决策变量：分时段出库流量或闸门开度方案；配水分配量；必要时的规则放宽标记与人工会商选择。
- 扰动因素：来水不确定性、降雨过程与预报误差、用水需求变化、工程状态变化、下游边界与潮位回水等。
- 约束条件：防洪限制水位与汛限要求、库容曲线与泄流能力、生态下泄与供水下限、下游断面控制、机组与闸门启闭约束、调度规程与会商规则。
- 时间尺度：短期滚动为主（例如小时到日尺度用于洪水过程与出库计划）；中期用于供水与调度计划（例如日到旬）；长期用于情景评估与水资源配置（例如月到季，第一阶段可先以情景方式承接）。
- 空间边界：多站点来水输入边界与站网覆盖范围；调度对象的单库/多库边界；下游控制断面与供水受水区边界。
- 评价指标：预报类（峰值误差、峰现时间、趋势一致性、预警提前量、RMSE等）；调度类（约束违反次数与幅度、防洪安全指标、供水保证率与缺水率、弃水与溢洪风险、方案可执行性）；系统类（滚动更新时效、可回放复现、解释与审计可追溯）。
- 交付边界：第一阶段交付“可跑通的滚动预报与规则调度原型服务+评估与预警机制”，不承诺最终最优调度或全量高精度模型体系。

## 二、最低实现方案

### 1. 实现目标

1. 形成统一口径的数据接入与回放框架，支持按固定步长滚动产出未来窗口的多站点来水预测结果与置信信息。
2. 形成规则约束下的滚动调度仿真与方案比选能力，输出分时段出库计划与关键约束校核结果。
3. 形成基础风险分析与预警触发能力，至少支持洪水预警与干旱预警的阈值触发、提前量统计与事件回放。
4. 形成预测质量与误差识别能力，支持滑动窗口评估、基于阈值的误差异常标记，并输出可解释的误差归因摘要。
5. 形成集合预测封装能力，至少支持加权平均与贝叶斯平均两类集成接口，并支持权重随滑动窗口更新。

### 2. 实现范围

- 纳入  
  
  - 数据分析：趋势、周期、突变的基础统计分析与可回放报告输出  
  - 预测：短中期滚动多站点径流预测的基线模型与集成封装；长期先以情景或统计分布承接  
  - 风险：入库不确定性与关键指标超限概率的基础统计  
  - 误差解释：滑动窗口评估、阈值异常识别、典型事件对比与可读结论输出  
  - 预警：洪水与干旱的阈值触发、分级规则、提前量统计与事件管理  
  - 调度：规程规则曲线配置化、滚动仿真、方案比选、约束校核与出库计划输出  
  - 配置：供需约束下的简化配水规则或启发式分配，并与调度仿真耦合展示

- 不纳入  
  
  - 多库联合全局最优求解、复杂大规模随机优化或高频闭环最优控制  
  - 雷达、卫星、同化全融合的高复杂度预报体系  
  - 全量数字孪生平台化底座与大规模数据治理改造  
  - 面向生产的指令下发与回执闭环的全链路控制系统（可预留接口）

### 3. 实现路径

- 预测侧最低路线：先建立可解释的基线模型与统一接口（概念模型或简化统计模型），再做误差订正与集合封装；集合层优先实现加权平均与贝叶斯平均，权重由滑动窗口历史表现更新；趋势一致性检验用于质量门控与模型切换提示。
- 风险与预警侧最低路线：以阈值与规则为主，配合概率统计输出超限概率与风险等级；预警触发先做规则白盒化与回放评估，再考虑更复杂的概率预警。
- 调度侧最低路线：先把现行规程、规则曲线与工程边界配置化，并建立滚动仿真与方案比选；仅在简单规则明显无法满足时，再升级到低复杂度优化或启发式改进。
- 解释与异常侧最低路线：以“误差状态识别+事件对比+可读结论”为目标，先做阈值异常与滑动窗口漂移识别，再逐步扩展到更复杂的可解释性方法。

### 4. 运行场景

- 运行对象：单库或多库水库群的滚动预报与调度一体化原型；站点级来水预报与下游断面校核。
- 输入：实时水雨工情、历史水文序列、站点与工程元数据、库容曲线与泄流能力、调度规程规则、供需情景与阈值规则、模型配置与权重策略。
- 输出：多站点来水预测序列与置信信息；风险指标与超限概率；干旱/洪水预警事件与分级；滚动出库计划与约束校核结果；误差异常标记与解释摘要；配水分配结果与缺口指标。
- 执行周期：按固定步长滚动运行（例如小时或日）；支持会商按需触发情景比选。
- 滚动窗口：预测侧输出未来窗口；评估侧以滑动窗口更新权重与识别漂移；调度侧以滚动窗口生成分时段出库计划并逐次修正。
- 典型工况：汛期洪水过程与超汛限风险；枯水期保供与缺水风险；来水突变与数据缺测降级；规则放宽的会商情景比选。

### 5. 验证边界

- 本阶段可以验证什么  
  
  - 预报服务的滚动产出、时效与回放复现能力  
  - 基线模型与集合封装的可用性、趋势与峰现相关指标的基本表现  
  - 阈值预警规则的触发逻辑、提前量统计与历史事件回放效果  
  - 规则调度仿真的约束满足、方案比选与可执行性输出  
  - 误差异常识别与解释摘要在典型事件上的可读性

- 本阶段暂时不能验证什么  
  
  - 在多库联合、复杂工况下的全局最优性或长期收益最优  
  - 高精度同化融合或端到端智能模型在全场景的稳健性  
  - 生产级指令闭环与现场执行一致性  
  - 无充分历史样本时的概率风险结论的高可信度

## 三、数据与伪数据方案

### 1. 最低数据需求

- 必需数据  
  
  - 历史与实时水文：雨量、水位、流量、入库/出库、蒸发等（按业务变量确定）  
  - 站点与流域元数据：站码、位置、汇流关系或分区关系、率定曲线版本  
  - 工程与能力：库容曲线、闸门机组泄流能力与状态枚举、关键断面控制信息  
  - 规程规则：汛限水位、调度曲线、硬约束与可放宽规则、预警阈值与分级规则  
  - 供需与配置：用水需求序列或情景、供水对象清单、优先级与最低保障约束

- 可经验给定数据  
  
  - 安全余量与经验限制、典型洪水事件清单、典型枯水过程清单  
  - 预警提前量要求、业务评价指标权重与门限  
  - 规则放宽条件与人工会商边界

- 可由伪数据替代的数据  
  
  - 来水情景与误差场景（不同峰值、峰现时间、持续时长）  
  - 闸门受限或机组故障情景、下游约束收紧情景  
  - 需求突增突降与缺水情景、缺测与观测噪声情景

### 2. 数据口径要求

- 字段与单位：明确水位、流量、雨量、库容、出库、需求量等字段及单位，明确率定曲线与库容曲线版本标识字段。
- 时间粒度：明确最小时间步长与对齐规则，预测更新步长与调度步长是否一致；明确时区与时间戳规范。
- 空间对象：站点编码唯一性、库区与断面对象编码、供水对象编码；多站点汇总口径与空间边界定义。
- 更新频率：实时数据到达频率与延迟容忍；滚动发布频率；规则与曲线版本更新频率。

### 3. 伪数据构造方案

- 构造对象：多站点来水过程、库水位与库容初始状态、泄流能力与状态枚举、供需序列、预警阈值与规则集合。
- 关键字段：时间戳、站点/对象编码、变量值与单位、版本号、缺测标记、质量标记、情景编号。
- 生成规则：  
  - 来水情景按峰值、峰现时间、持续时长、底水抬升与突变程度分组生成，并叠加可控误差场景  
  - 供需情景按季节性周期与突发扰动生成，并生成缺水等级  
  - 工程状态按可执行枚举生成，并生成受限与故障子集
- 典型工况覆盖方式：覆盖汛期洪水、枯水保供、突变与缺测、规则放宽会商四类主场景，并保证每类场景可回放复现。

本阶段可用伪数据验证：接口契约、滚动机制、规则触发、仿真与比选链路、异常识别链路。  
本阶段不能仅靠伪数据证明：模型在真实数据噪声、真实工程执行与真实业务策略下的最终精度与最优性。

### 4. 后续数据整理要求

后续数据清洗整理目标是向前期伪数据规范对齐。  

- 字段对齐：站点与对象编码统一、变量命名统一、规则与曲线版本字段补齐。  
- 单位对齐：雨量、水位、流量、库容、供需量等单位统一并建立可追溯的换算记录。  
- 粒度对齐：统一时间步长与对齐策略，明确聚合与插值原则，保留原始与处理后两套序列。  
- 缺失值处理：明确缺测标记、降级策略与插补策略的适用边界，并记录处理日志。  
- 异常值处理：建立质量标记与阈值规则，区分观测异常、传输异常与真实突变，并保留可回放证据。

## 四、待确认事项

### 1. 需反馈甲方确认事项

- [ ] 调度目标优先级与权衡口径：防洪、保供、发电、生态的优先级是否固定，是否允许会商调整  
- [ ] 调度规则的硬约束与软约束边界：哪些规则不可违反，哪些可在会商下放宽  
- [ ] 调度对象边界：先做单库还是多库，是否涉及联合调度与上下游耦合  
- [ ] 预报输出口径：输出变量是来水、流量、水位还是多变量；短中长期预见期与更新频率要求  
- [ ] 预警规则口径：干旱与洪水的阈值、分级、提前量要求、发布流程与审核机制  
- [ ] 验收口径：预报与预警采用哪些业务指标为主，调度方案以哪些约束与绩效指标验收  
- [ ] 数据接口与权限：实时数据接入方式、历史数据可追溯范围、规程版本与更新权限  
- [ ] 控制权限边界：是否需要指令下发与回执闭环，异常时人工接管流程

### 2. 需执行方内部确认事项

- [ ] 第一阶段最小可交付边界：优先交付“规则白盒化+滚动仿真+预报服务”还是“优化求解”  
- [ ] 时间步长与计算频率：小时级还是日级作为主运行粒度，滚动窗口长度与更新频率  
- [ ] 基线模型选择：概念模型、简化统计模型或两者并行；集合层的权重更新策略与默认参数  
- [ ] 缺测与低质量数据降级策略：缺测时输出如何降级，预警与调度如何保守运行  
- [ ] 评估指标体系：技术指标与业务指标的最小集合，以及指标计算的口径与版本管理  
- [ ] 调度规则表达方式：规则曲线与条款如何配置化、如何审计与回放，版本控制策略  
- [ ] 原型系统集成方式：数据接入、服务编排、结果落库与可视化最小闭环的接口划分  
- [ ] 风险接受度：第一阶段允许的精度与误报漏报边界，允许的人工会商介入频率

## 五、下一步工作建议

### 1. 近期推进建议

1. 组织一次内部“问题定型与边界确认会”，输出：调度对象边界、预报预见期与更新频率、预警阈值口径、第一阶段验收指标最小集合。
2. 用伪数据先跑通端到端链路：多站点预测接口 → 集合封装 → 风险与预警触发 → 规则调度仿真与比选 → 结果回放报表。
3. 并行推进两项底稿：规程规则条款清单与配置化表达草案；数据字典与时间粒度对齐规则草案。
4. 选定2到3个典型历史事件作为第一阶段回放基准集，用于验证预警触发与调度约束校核链路。

### 2. 进入下一阶段前需补充内容

- 真实数据可得性与质量评估结论：缺测比例、延迟、异常类型分布与可用时段覆盖
- 规程规则的正式版本与变更机制：条款解释权、放宽流程、版本审计要求
- 调度与配置的业务流程：会商频率、审批链路、输出指令格式与责任边界
- 预警业务规则细化：分级处置措施、阈值调整机制、误报漏报容忍边界
- 多库联合与优化升级条件：何时需要从规则仿真升级到简单优化或更复杂方法的触发标准
